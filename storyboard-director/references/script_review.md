# Script Review

## Purpose

Use this file before writing shots.

The goal is to review the script as a visual storytelling problem, not just to restate the plot.

## Required review questions

For each scene or beat, answer:

- what is the beat trying to express
- what must the audience understand without explanation
- what must be seen, not just implied
- where would fast pacing create confusion
- where should the scene slow down for readability
- what action chains require setup, result, and reaction
- what emotional changes must be visible on the character
- what object states must remain controlled

## Red flags

Revise before shot writing if any of these are true:

- the beat depends on a failure or success that is not visually defined
- two characters act in sequence but the handoff is not clear
- the scene depends on expression but expression is not described
- the object state is important but not specified
- the written timing is shorter than the readable action chain

## Output of review

For each scene, produce:

- `scene_objective`
- `audience_must_understand`
- `readability_risk`
- `expression_risk`
- `object_state_risk`
- `timing_risk`
- `recommended_strategy`

Use this review internally before final prompt writing.
