# 输出契约

默认最终输出只包含分镜本身，不包含过程分析、评分表、内部 JSON 或术语说明。

## 格式

每个镜头必须写成：

```text
01A [景别｜持续时间]
机位/运动：...
画面：...
发生：...
接点：...
情绪：...
```

## 字段规则

### 镜号

- 以场次编号加字母表示，例如：
  - `01A`
  - `01B`
  - `02A`
- 同场戏内连续编号，不跳号。

### 景别

- 用中文标准称呼：
  - `大全景`
  - `全景`
  - `中景`
  - `双人中景`
  - `中近景`
  - `近景`
  - `特写`
  - `插入特写`
- 不要在景别栏塞风格形容词。

### 持续时间

- 使用 `X秒` 或 `X.X秒`
- 默认写易执行的时长，不写虚词
- 连接镜通常更短，主镜可适当更长

### 机位 / 运动

- 只写对执行有帮助的内容：
  - 机位关系
  - 是否固定 / 推 / 拉 / 跟 / 摇
  - 是否需要守轴或主观视角
- 不要写空洞形容词，比如“很电影感”“高级”。

### 画面

- 写观众真正会看到什么
- 优先交代：
  - 人物与关键物件
  - 空间关系
  - 视觉焦点
- 不要把“发生”里的动作整段重复一遍

### 发生

- 写这一镜里发生的动作和信息推进
- 如果有台词，只保留必须帮助分镜执行的那一句或概述其作用

### 接点

- 每镜必写
- 接点优先写“为什么这样切”：
  - 动作接切
  - 视线引切
  - 信息揭示切
  - 反应切
  - 节奏收住
- 不要只写“切到下一镜头”

### 情绪

- 用简洁中文概括镜头落点
- 写对表演和节奏有帮助的词：
  - 克制
  - 压抑
  - 紧绷
  - 试探
  - 决绝
  - 松动

## 语气

- 专业、直接、可执行
- 不做教学口吻
- 不做散文式修辞

## 禁止输出

- 不要展示内部分析字段
- 不要写“以下是我的分析”
- 不要给多个分镜方案，除非用户明确要求
- 不要新增原脚本没有的新剧情事实
