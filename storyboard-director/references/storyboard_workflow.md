# Storyboard Workflow

## Purpose

Use this file to turn script understanding into storyboard shots.

## Pipeline

1. identify the scene objective
2. identify what the audience must understand
3. identify the protagonist hierarchy
4. identify expression-critical beats
5. identify state-critical objects
6. map the beat into 1-5 shots
7. validate action timing
8. validate expression readability
9. validate object state
10. critique and score
11. render storyboard prompt lines

## Scene objective

Before shot planning, name the scene objective in one line.

Examples:

- establish the problem
- reveal the helper
- show the first successful change
- release wonder
- land the gift
- close on brand memory

If the objective is vague, the shots will be decorative.

## Shot count heuristic

Prefer fewer stronger shots.

- short emotional beat: `1-3` shots
- standard action beat: `2-5` shots
- promo/ad beat: `2-4` shots
- long take: allowed, but still validate internal action readability

## One shot, one main job

A shot may contain several micro-actions, but it should only carry one primary dramatic job.

Good:

- attempt and visible failure
- helper arrives
- first magical success
- reaction and release

Bad:

- establish problem + fail twice + helper appears + solution reveal all in one shot

## Expression and state checkpoints

For each shot, ask:

- what the face or body is communicating
- whether the audience can read the feeling change
- whether any object must stay in a specific state

If either performance or object state is critical, make it explicit in the shot plan.

## Long-take rule

Use a long take only if:

- the action chain remains readable
- the camera move helps the handoff between beats
- object state remains visually clear
- expression changes are still legible

If a long take tries to carry too many story turns, split it.
